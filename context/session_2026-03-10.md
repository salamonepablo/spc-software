### 4. DbContext Updates

Added explicit EF Core relationship configurations in `SPCDbContext.cs`:

```csharp
// Invoice relationships
modelBuilder.Entity<Invoice>()
    .HasOne(f => f.Customer)
    .WithMany(c => c.Invoices)
    .HasForeignKey(f => f.CustomerId);

modelBuilder.Entity<Invoice>()
    .HasOne(f => f.SalesRep)
    .WithMany(v => v.Invoices)
    .HasForeignKey(f => f.SalesRepId);

modelBuilder.Entity<InvoiceDetail>()
    .HasOne(d => d.Invoice)
    .WithMany(f => f.Details)
    .HasForeignKey(d => d.InvoiceId);

modelBuilder.Entity<InvoiceDetail>()
    .HasOne(d => d.Product)
    .WithMany()
    .HasForeignKey(d => d.ProductId);
```

### 5. Blazor Fix

Fixed `NewInvoice.razor` compilation error on line 67:

**Before (error):**
```razor
<input type="number" @bind="item.Quantity" @bind:event="oninput"
       @onchange="@(null)" min="1" step="1" class="form-control" />
```

**After (fixed):**
```razor
<input type="number" @bind="item.Quantity" @bind:event="oninput"
       value="" min="1" step="1" class="form-control" />
```

The error was caused by Blazor's inability to determine if `@(null)` was a string null or delegate null.
