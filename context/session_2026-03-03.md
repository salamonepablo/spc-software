## Key Design Decisions

### 1. Response DTOs with Flattened Relations
Instead of returning nested objects, response DTOs flatten related entity data:
```csharp
public class ClienteResponse
{
    // ... basic properties ...
    
    // Flattened from CondicionIva navigation
    public int? CondicionIvaId { get; set; }
    public string? CondicionIvaDescripcion { get; set; }
    public string? TipoFactura { get; set; }
    
    // Flattened from Vendedor navigation
    public int? VendedorId { get; set; }
    public string? VendedorNombre { get; set; }
}
```

### 2. Service Responsibility
Services handle:
- Database interaction (EF Core)
- Entity в†” DTO mapping
- Business rules
- Navigation property loading

Endpoints only:
- Receive HTTP request
- Call service
- Return appropriate HTTP result

### 3. Endpoint Grouping
Using `MapGroup` for cleaner route organization:
```csharp
var group = app.MapGroup("/api/clientes")
    .WithTags("Clientes");

group.MapGet("/", ...);
group.MapGet("/{id:int}", ...);
```

### 4. Service Registration
Services registered as Scoped (one instance per request):
```csharp
builder.Services.AddScoped<IClientesService, ClientesService>();
builder.Services.AddScoped<IProductosService, ProductosService>();
```
