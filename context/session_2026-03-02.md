## Technical Decisions

### Why InMemory Database for Tests?
- Faster execution (no SQL Server startup)
- Isolated per test class
- No external dependencies
- Easy to seed test data

### Why Manual Seed Data?
- EF Core's `HasData()` only works with migrations
- InMemory provider doesn't run migrations
- Manual seeding in `CreateHost()` ensures data exists

### Why Skip Migrate() in Testing?
- `Database.Migrate()` is relational-specific
- Throws exception with InMemory provider
- Conditional check: `if (!app.Environment.IsEnvironment("Testing"))`

### Why Unique Prefixes in Search Tests?
- Tests in same class share database
- Data from one test persists to next
- Unique prefixes ensure test isolation
