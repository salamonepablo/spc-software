# Session: Factura A vs B Business Rules Implementation

**Date:** 2026-03-08

## Goal

Implement CRUD endpoints for Invoices (Facturas), Credit Notes, Debit Notes, and Quotes (Presupuestos) with complete business rules for pricing, discounts, and VAT calculations following Argentine tax regulations.

Key business rules:
1. **Factura A**: Net prices (PrecioFactura) + VAT discriminated + IIBB perception if company is agent
2. **Factura B**: Final prices with VAT included (PrecioPresupuesto), shows "IVA Contenido" (Ley 27.743)
3. **IIBB Perception**: Only calculated if company is "Agente de Percepción" AND customer has AlicuotaIIBB > 0 from AFIP padrón
4. **Multi-level discounts**: Customer default → Document level → Line level
5. **VAT immutability**: Store VAT% in document at creation time for historical integrity

- `IIBBProvince` - Province where company is registered

### 2. Cliente Updated with IIBB Data

Added fields to `Cliente` for AFIP padrón integration:
- `AlicuotaIIBB` - IIBB perception rate from ARBA/AGIP padrón
- `ProvinciaPadronIIBB` - Province of the padrón

### 3. Dual Pricing on Products (Updated)

Added two price fields to `Producto`:
- `PrecioFactura` - Invoice price (without VAT)
- `PrecioPresupuesto` - Quote price (with VAT included)

### 4. Factura Entity Updated

Added `IVAContenido` field to store VAT included amount for Factura B (Ley 27.743).

### 5. Pricing Service Enhanced

Updated `PricingService` with two calculation methods:

**CalculateDocumentTypeA():**
- Uses net prices (PrecioFactura)
- Adds VAT to subtotal
- Adds IIBB perception if company is agent AND customer has AlicuotaIIBB > 0

**CalculateDocumentTypeB():**
- Uses final prices (PrecioPresupuesto, VAT included)
- Calculates "IVA Contenido" = Total / 1.21 * 0.21
- Does NOT add additional VAT (already included in price)

### 6. FacturasService Updated

Updated `CreateAsync` to handle Factura A vs B:
- Selects correct price field based on document type
- Calls appropriate pricing method
- Stores correct values in database

### 7. Tests Fixed and Expanded

Fixed tests that were expecting Factura A calculations while using Factura B. Total **111 tests passing**.
